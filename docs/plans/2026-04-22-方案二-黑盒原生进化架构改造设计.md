# 方案二黑盒原生进化架构改造设计

## 概述

基于现有系统，渐进式改造为方案二：单工具黑盒原生进化架构。

### 核心原则

- **保留现有代码**：React前端 + FastAPI后端
- **渐进式增强**：非破坏性改造
- **体验优先**：保持用户体验连续

## 核心组件设计

### 1. 完整人格提示词系统

创建多语言（中文+英文）的完整人设模板，包含：
- 系统身份定义
- 讲解原则（由浅入深、文化共情、语言优雅）
- 边界规则（不能做的事情）
- "不知道"的优雅处理方式
- 动态人格微调（根据用户对话历史）

### 2. 最小可行版围栏机制

**第一级：输入过滤**
- 简单关键词过滤（政治敏感词、恶意输入）
- 基础内容分级判断

**第二级：人设强化**
- 每次对话前动态注入核心人设
- 历史上下文清洗（移除无关或破坏人设的内容）
- "不知道"的优雅处理模板注入

**第三级：输出审核**
- 基础事实一致性检查（简单关键词匹配）
- 语气风格一致性检查（简单正则）
- 违规检测基础版

### 3. 完整进化机制

**例子库数据结构：**
```json
{
  "id": "uuid",
  "type": "good_example|bad_example",
  "question": "用户问题",
  "exhibitId": "展品ID",
  "answer": "系统回答",
  "rating": 1-5,
  "notes": "人工审核备注",
  "createdAt": "时间",
  "version": "v1.0"
}
```

**版本管理：**
- 简单的JSON配置文件版本化
- 支持手动标记"稳定版"
- 支持一键回滚到上一个稳定版本

## 数据流设计

### 完整对话流程

```
1. 用户输入
   ↓
2. 输入过滤（第一级围栏）
   - 关键词检查
   - 内容分级
   ↓
3. 会话状态获取
   - 获取历史记录
   - 获取当前展品
   - 获取当前深度
   ↓
4. 构建提示词
   - 注入核心人设
   - 注入例子库（3-5个相关例子）
   - 注入展品本体数据
   - 注入历史上下文
   ↓
5. 调用Moonshot K2
   ↓
6. 输出审核（第二级+第三级围栏）
   - 事实一致性检查
   - 语气风格检查
   - 违规检测
   ↓
7. 失败：降级处理（使用本体数据预设内容）
   成功：返回给用户
   ↓
8. 自动提取候选例子
   ↓
9. 加入人工审核队列
   ↓
10.（可选）人工审核后加入正式例子库
```

## API和数据结构设计

### API变更

现有API保持不变，新增几个管理API：
- `GET /api/examples` - 获取例子库列表
- `POST /api/examples` - 添加新例子（管理员）
- `PUT /api/examples/{id}` - 审核例子
- `GET /api/versions` - 人格版本列表
- `POST /api/versions/rollback` - 回滚版本

### 数据目录结构

```
data/
  exhibits.json          # 现有展品本体数据
  persona/

      v1.1.json
  examples/
    good.json            # 好例子库
    bad.json             # 坏例子库
    pending.json         # 待审核队列
```

```
  evolution.py           # 进化机制模块（新）

```
