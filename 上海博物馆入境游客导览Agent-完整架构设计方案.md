# 上海博物馆入境游客导览 Agent - 完整架构设计方案

> 两套架构完整对比 + 治理机制 + 团队心态建设指导

---

## 📋 项目背景

针对上海博物馆入境游客（主要是欧美游客），打造沉浸式、高人文素养的 AI 导览 Agent。

**核心挑战：**
- 不是简单的知识问答，而是要有历史深度、文化共情、语言优雅的叙事能力
- 需要处理边界情况（不知道答案、政治敏感话题、用户情绪）
- 体验优先级高于功能完整性

## 方案一：多工具 Hermes 确定性架构

### 核心理念

**人机分工边界清晰：** 人类做内容和规则设计，LLM 只做润色和衔接，绝不做事实决策。

### 架构分层

```
┌─────────────────────────────────────────────────────────┐
│                     用户交互层                            │
│              微信 / App / Web 前端                        │
└─────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────┐
│                    Hermes Agent 编排层                    │
│          会话管理 ── 工具路由 ── 状态跟踪                  │
└─────────────────────────────────────────────────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  事实检索工具    │  │  叙事生成工具    │  │  边界处理工具    │
│  文物知识图谱    │  │  人格模板引擎    │  │  熔断机制       │
│  历史事件库     │  │  语言润色       │  │  降级策略       │
└─────────────────┘  └─────────────────┘  └─────────────────┘
          │                   │                   │
          └───────────────────┼───────────────────┘
                              ▼
┌─────────────────────────────────────────────────────────┐
│                     知识库层                              │
│          文物本体 ── 历史本体 ── 人格本体                 │
└─────────────────────────────────────────────────────────┘
```

### 核心优势

1. **100% 事实准确性：** 所有事实回答来自预构建的知识图谱，LLM 只负责语言润色
2. **完全可审计：** 每一步工具调用都有日志，问题可追溯
3. **边界可控：** 五级围栏机制确保不会出界
4. **渐进式进化：** 例子库持续积累，系统稳步变好

### 核心劣势

1. **体验上限较低：** 很难产生超越预期的"哇时刻"
2. **工具切换可能生硬：** 用户能感觉到系统在不同模式间切换
3. **维护成本较高：** 需要持续维护知识图谱和工具链

## 方案二：单工具黑盒原生进化架构

### 核心理念

**接受黑盒不可控性，在边界建秩序而非试图控制黑盒内部。**

### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                     用户交互层                            │
└─────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────┐
│                    边界围栏层                             │
│          输入过滤 ── 输出审核 ── 异常熔断                  │
└─────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────┐
│                    黑盒核心层                             │
│          GPT-4o / Claude 3 Opus 原生能力                   │
│          + 完整人设系统提示词                              │
│          + 动态注入的例子库                                │
└─────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────┐
│                    进化机制层                             │
│          例子自动提取 ── 人工审核 ── 版本发布               │
└─────────────────────────────────────────────────────────┘
```

### 核心优势

1. **体验天花板极高：** 有涌现惊喜的可能性，可能产生真人导游都做不到的体验
2. **实现简单：** 不需要复杂的工具编排和知识图谱
3. **自然流畅：** 用户完全感觉不到是在和 AI 对话

### 核心劣势

1. **不可控性：** 永远存在"胡说八道"的风险，只能降低不能消除
2. **可审计性差：** 出了问题很难追溯原因
3. **团队心态挑战：** 需要接受"偶发异常是正常现象"

---

## 🛡️ 五级围栏治理机制

两套架构共享同一套围栏体系：

### 第一级：输入过滤

- 政治敏感词检测
- 明显恶意输入拦截
- 内容分级判断

### 第二级：人设强化

- 每次对话前动态注入核心人设
- 历史上下文清洗
- "不知道"的优雅处理模板

### 第三级：输出审核

- 事实一致性检查
- 语气风格一致性检查
- 边界规则违规检测

### 第四级：异常熔断

- 连续 3 次违规自动降级
- 用户情绪检测（愤怒/沮丧）触发人工介入
- 严重违规直接终止会话

### 第五级：一键回滚

- 所有人设版本化管理
- 发现问题一键回滚到上一个稳定版本
- 灰度发布机制，新特性只对 10% 用户生效

## 🔄 自动进化机制

### 例子库自动提取

```
用户会话 → 自动标记 → 人工审核 → 加入例子库
```

- 好的例子：回答特别精彩，用户特别满意
- 坏的例子：回答出错，用户不满意，边界处理不当
- 每个例子标注：问题类型、处理方式、效果评分

### 例子库动态注入

- 每次对话根据用户问题类型，动态注入 3-5 个最相关的例子
- 例子库按使用频率和效果评分自动排序
- 例子库定期清理，移除过时或效果不佳的例子

### 人格版本管理

- 每周发布一个新的人格版本
- 包含：新增例子、优化的提示词、边界规则更新
- 所有变更经过团队评审
- 版本发布后有 24 小时观察期，异常则一键回滚

## 🧠 团队心态建设指导

### 必须接受的现实

1. **偶发异常是正常现象：** 追求零风险 = 零创新。接受 0.1% 的异常率，换来 200% 的体验上限。

2. **渐进式退化而非爆发式进化：** 不要期待"训练一次就永久完美"，系统是每天变好一点点，而不是某天突然质变。

3. **诚实比完美重要：** 不知道就明确说不知道，用预设的方式优雅处理边界问题，用户会理解；强行编造答案才是真正的体验杀手。

### 反人性模仿原则

- 不试图在人类擅长的事情上超越真人（比如情感深度、即兴幽默）
- 而是把机器的超能力发挥到极致（比如记住每件文物的所有细节、无缝切换 10 种语言、24 小时不疲劳）

### 体验优先原则

> 宁可少讲 10 个知识点，也不能破坏用户的沉浸感。

## 📊 推荐实施路径

### 第一阶段（第 1-2 周）：双架构并行开发

- 团队分成两组，同时开发两套架构
- 共享同一套围栏机制和例子库
- 每周五做一次体验对比评测

### 第二阶段（第 3-4 周）：灰度发布

- 两套系统同时上线，各分配 50% 流量
- 数据收集：用户满意度、会话时长、边界触发次数
- 每周团队评审，根据数据调整架构

### 第三阶段（第 5-12 周）：数据驱动收敛

- 运行 2 个月后根据数据决定最终架构
- 很可能的结果：保留黑盒核心 + 完善的围栏体系
- 围栏体系本身会成为核心竞争力

## 🎯 核心决策点总结

| 决策项 | 建议 |
|--------|------|
| **核心架构** | 双架构并行灰度，数据驱动 |
| **围栏机制** | 五级围栏，必须完整实现 |
| **进化机制** | 例子库自动提取 + 人工审核 |
| **版本管理** | 人格版本化，一键回滚 |
| **团队心态** | 接受偶发异常，控制概率而非追求零风险 |
| **体验原则** | 诚实比完美重要，沉浸感优先于信息量 |
